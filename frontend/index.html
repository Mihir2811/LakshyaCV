Professional Summary

2–3 sentences that sell you before anyone reads further. Recruiters spend roughly 6 seconds on a resume.

Summary tips

  • Open with your strongest skill or years of experience — not “I am a…”
  • Name the type of role or industry you are targeting. Show intent.
  • Include one achievement with a number: “reduced load time by 40%”.
  • Avoid clichés: “passionate”, “hardworking”, “team player” — everyone says this.
  • Example: “Software engineer with 3 years building scalable Django APIs. Reduced deployment time by 60% through CI/CD automation. Seeking a backend role in fintech.”

Skills

Comma-separated. Rendered as pill tags in a row-based grid on your PDF.

Skills tips

  • List hard skills first: languages, frameworks, tools, platforms.
  • Mirror keywords from the job description — ATS scanners look for exact matches.
  • Aim for 10–16 skills. Too few looks sparse; too many looks unfiltered.
  • Skip “Microsoft Word” unless it is explicitly required.
  • Soft skills belong in your summary or bullet points, not here.
e.g. Python, Django, React, PostgreSQL, Docker, Git

Work Experience

Most recent role first. One achievement per line. Start each line on a new line.

Experience tips

  • Use the CAR formula: Challenge → Action → Result.
  • Start every bullet with a strong verb: Built, Led, Reduced, Shipped, Designed.
  • Quantify everything: “reduced API response time by 35%”, “managed 4 engineers”.
  • 3–5 bullets per role is ideal. More dilutes impact.
  • Write achievements, not duties. “Responsible for X” → “Delivered X resulting in Z”.
  • Avoid first person: not “I built…” — just “Built…”

Education

Most recent degree first. Include certifications and online courses here too.

Education tips

  • Use the full degree name: “Bachelor of Science in Computer Science”.
  • Include AWS, Google Cloud, Coursera certifications here too.
  • List relevant coursework only if you are a recent graduate with limited experience.
  • Do not include high school if you have a university degree.
  • Do not include GPA below 3.0 — leave it blank.

Projects

Side projects, open source, and academic work all count. 2–3 strong projects beat 6 weak ones.

Project tips

  • Describe the problem it solves: “An app that reduces X” beats “A web app”.
  • Include a live URL or GitHub link — shown as “View” hyperlink in the PDF.
  • Mention scale or traction: “used by 200+ students”, “500 GitHub stars”.
  • List the specific tech stack — recruiters and engineers scan for this.
  • Do not list tutorial projects unless heavily extended.
